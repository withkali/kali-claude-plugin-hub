virtual-team

서비스 아이디어를 받아 기획→개발→(운영)까지 자동화하는 가상 개발팀 플러그인.

설치

/plugin marketplace add withkali/kali-claude-plugin-hub
/plugin install virtual-team@kali-claude-plugin-hub

슬래시 명령

명령설명
/new-service <아이디어>새 서비스의 Phase 1(기획 정합)을 시작. slug·기획서 생성, 파트 질의 수집, 보강, 운영자 승인 요청.
/team-roster역할 로스터와 모델/단계 정책 출력.
/activate-phase3 <slug>해당 제품에 테스트·배포·모니터링·장애관리 단계를 편입.

워크플로우

  • Phase 1 — 기획 정합: 아이디어 → planner 기획서 → 파트 질의 → 보강 → 운영자 승인.
  • Phase 2 — 개발: (필요 시 infra가 사전 요구사항 질의서 작성) → 각 파트 개발, 설계는 opus·구현은 sonnet → PR.
  • Phase 3 — 운영: 테스트·배포·모니터링·장애관리. 기본 비활성, 운영자가 활성화 시 편입.

기록(Records)

모든 파트는 작업·질의/답변을 작업 디렉토리의 records/<slug>/에 남기고, 외부 산출물(GitHub Issue/PR)은 링크로 연결합니다. 로컬 파일은 "무엇을·어디서 확인하는지"의 인덱스입니다.

보안 가드레일

  • 키·credential 하드코딩 금지 — env/시크릿 매니저로만 주입, 저장소엔 .env.example만.
  • Write/Edit는 동봉된 secret-scan 훅이 자동 검사·차단(개인키·토큰·AWS/GCP 키·실제 .env 등).
  • 클라/웹 하드코딩이 불가피하면 중단·보고·승인(백엔드 프록시·키 제한·회전 등 완화책 동반).

권한 · 자율성

권한 allow/deny는 플러그인이 아니라 settings.json에 둡니다. 권장(균형): 로컬·가역 작업(읽기/쓰기/커밋/빌드/테스트)은 자동, 원격 push·배포·prod/DB 변경·파괴적 명령·시크릿 하드코딩은 운영자 승인.

한 번 승인한 위험 작업이 항상 승인된 것은 아닙니다 — 범위가 바뀌면 다시 확인합니다.
