virtual-team

v0.1.0 · MIT · withkali
22 역할 에이전트 3 슬래시 명령 보안 훅 opus · sonnet · haiku

러프한 한 줄 아이디어를 던지면, 기획자가 기획서를 쓰고 → 디자이너·개발자들이 질의로 다듬고 → 승인 후 개발에 들어갑니다. 모든 파트가 자기 작업과 질의·답변을 기록으로 남깁니다.

주요 기능

🧩

22개 역할 자동 분업

오케스트레이터가 작업을 쪼개 적합한 파트에 위임하고 결과를 종합·보고합니다.

🧠

작업 유형별 모델

검색=haiku, 실행=sonnet, 기획·설계·분석=opus로 자동 라우팅.

🔒

시크릿 차단 훅

키·토큰·실제 .env가 코드에 들어가면 Write/Edit 단계에서 자동 차단.

🚦

단계적 워크플로우

기획→개발은 활성, 테스트·배포·장애(Phase 3)는 승인 시 편입.

슬래시 명령

/new-service <아이디어>Phase 1 시작 — slug·기획서 생성, 파트 질의 수집, 보강, 승인 요청
/team-roster역할 로스터와 모델·단계 정책 출력
/activate-phase3 <slug>테스트·배포·모니터링·장애관리 단계 편입

워크플로우

  • Phase 1 · 기획 정합 — 아이디어 → 기획서 → 파트 질의 → 보강 → 운영자 승인
  • Phase 2 · 개발 — (필요 시 인프라 사전 요구사항 질의) → 설계(opus)·구현(sonnet) → PR
  • Phase 3 · 운영 — 테스트·배포·모니터링·장애관리 (기본 비활성)

보안 · 자율성

  • 키·credential 하드코딩 금지 — env/시크릿 매니저로만, 저장소엔 .env.example만.
  • 로컬·가역 작업은 자동 진행, 원격 push·배포·prod 변경·시크릿 하드코딩은 운영자 승인.
권한 allow/deny는 플러그인이 아니라 settings.json에 둡니다. 확인 프롬프트를 줄이려면 전역 ~/.claude/settings.json에 균형 권한을 두세요.

설치

/plugin marketplace add withkali/kali-claude-plugin-hub
/plugin install virtual-team@kali-claude-plugin-hub
