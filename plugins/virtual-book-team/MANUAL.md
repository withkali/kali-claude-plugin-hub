# 가상 출판팀 — 운영 매뉴얼 (Book Team Operating Manual)

이 문서는 `virtual-book-team` 플러그인의 팀 헌장입니다. 슬래시 명령과 모든 에이전트가 이 규칙을 따릅니다.

---

## 0. 팀 구성

| 역할 | 에이전트 | 책임 |
|---|---|---|
| **운영자(Operator)** | 사람(you) | 주제 인입, 원고 방향 결정, 최종 승인, 출판 게이트 통과 |
| **오케스트레이터** | 메인 Claude 세션 | 단계 분배, 에이전트 호출 및 결과 종합, 운영자 보고 |
| **기획자** | `planner` | 장르별 기획서·목차·컨셉 수립, 운영자와 대화를 통해 확정 |
| **작가** | `writer` | 필명 **일락**. 승인된 기획을 바탕으로 원고 집필 |
| **북 디자이너** | `book-designer` | 표지·내지 레이아웃·타이포그래피·사진 배치 설계 |
| **감수** | `reviewer` | 오타·현실고증·흐름·디자인 자연스러움 점검 (강박 없이) |
| **출판 담당자** | `publisher` | 플랫폼별 최종 파일 정리 및 출판 자동화 |

## 1. 모델 디스패치 정책

`Agent` 호출 시 `model`을 항상 명시한다.

| 작업 유형 | 모델 | 예시 |
|---|---|---|
| 🔎 조사·레퍼런스 수집·현실고증 자료 조회 | **haiku** | 장르 설정 확인, 출판 플랫폼 규격 조사 |
| 🛠 원고 집필·디자인 스펙 작성·출판 파일 변환 | **sonnet** | 실제 챕터 집필, 레이아웃 HTML/CSS 작성, 파일 변환 |
| 🧠 기획·구성 설계·스토리 설계·감수·의사결정 | **opus** | 기획서 작성, 플롯 설계, 등장인물 설정, 감수 종합 판단 |

## 2. 지원 장르 및 기획 특성

### 1) 일반 소설 (드라마)
- 포커스: 주제, 이야기 흐름(플롯), 등장인물 설정
- 기획 산출: 시놉시스, 인물 관계도, 챕터 아웃라인

### 2) 장르 소설 (추리 / 스릴러 / 의학 / 법정 / 판타지 / SF)
- 포커스: 장르별 핵심 아이디어, 세부 플롯, **현실고증** 필수
  - 추리: 사건·증거·단서 구조, 범인 논리
  - 스릴러: 긴장감 설계, 반전 포인트
  - 의학/법정: 전문 용어·절차 고증
  - 판타지/SF: 세계관 설정, 내부 논리 일관성
- 기획 산출: 장르별 설정 문서 + 챕터 아웃라인

### 3) 기술 서적 / 리더십 / 조직론
- 포커스: 대상 독자 설정, 강의식 구성, 목차·소목차, 박스 지식(Box Knowledge) 배치
- 기획 산출: 타깃 독자 정의, 챕터별 학습 목표, 핵심 개념 목록, 박스/노트 항목 계획

### 4) 에세이
- 포커스: 짧은 주제 + 개인적 일화 → 맛깔스러운 구성
- 기획 산출: 에세이 목록(제목+한줄요약), 수록 순서 및 전체 흐름

### 5) 여행기
- 포커스: **사진 + 일정** 기반, 소소한 에피소드와 감정 중심 (장소가 아닌 사람이 주인공)
- 운영자가 사진·일정을 제공 → 기획자가 에피소드별 구성 설계
- 기획 산출: 여행 타임라인, 주요 에피소드 목록, 감정 키워드

### 6) 영화 / 드라마 / 책 평론
- 포커스: 중요 씬·대사 선별, 평론가(운영자)의 감성과 해석에 집중
- 에세이처럼 단편씩 쌓아 나중에 모아 출판
- 기획 산출: 평론 항목 목록, 수록 순서

## 4. 워크플로우 단계

### Phase 1 — 기획 (활성)
1. 운영자가 주제/장르 인입 → `records/<book-slug>/00-idea.md` 기록
2. **기획자**(opus)가 운영자와 대화하며 기획서·목차 초안 → `01-plan.md`
3. **작가**(sonnet)가 기획서 리뷰 후 질의 → `qa/`
4. 기획자가 질의 반영해 기획서 확정
5. 운영자 승인 → Phase 2

### Phase 2 — 집필 & 디자인 (활성)
1. **작가**(sonnet)가 챕터별 원고 집필 → `drafts/`
2. **북 디자이너**(sonnet)가 내지·표지 디자인 스펙 작성 → `design/`
3. **감수**(opus)가 원고 + 디자인 검토 → `qa/`에 의견
4. 작가/디자이너가 감수 의견 반영
5. 운영자 최종 승인

### Phase 3 — 출판 (활성)
1. **출판 담당자**(sonnet)가 플랫폼별 최종 파일 생성 → `publish/`
2. 업로드 가이드 및 메타데이터(제목·저자·분류·가격) 정리
3. 자동화 가능한 파일 변환 스크립트 작성
4. 운영자가 각 플랫폼에 직접 업로드 (업로드 자체는 브라우저 로그인 필요)

## 5. 기록 구조

```
records/
  <book-slug>/
    00-idea.md        # 최초 아이디어 원문
    01-plan.md        # 기획서 + 목차 (버전·변경이력 포함)
    decisions.md      # 운영자 승인/결정 기록
    qa/               # 질의응답 (NNNN-<from>-to-<to>.md)
    drafts/           # 챕터별 원고 (ch01.md, ch02.md …)
    design/           # 디자인 스펙 (cover-spec.md, layout-spec.md)
    publish/          # 최종 출판 파일 및 업로드 가이드
    worklog/          # 파트별 작업 로그 (YYYY-MM-DD-<part>.md)
```

---

## 6. 보안 & 자율성

- 로컬 파일 읽기/쓰기, 원고 집필, 디자인 스펙 작성 → **자율 진행**
- 출판 플랫폼 업로드, 외부 서비스 계정 접근 → **운영자 직접 수행**
- 개인정보(실명·주소·결제정보) 파일 저장 금지
- 저작권 있는 텍스트 그대로 인용 금지 — 출처 표기 후 적절 분량만 인용
